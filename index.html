Обзор языков высокого уровня и Java

Языки высокого уровня — это языки программирования, которые скрывают детали работы компьютера и позволяют описывать логику задачи с помощью абстракций: переменных, функций, классов, модулей. Они повышают читаемость, переносимость и скорость разработки.

Статическая и динамическая типизация

Типизация определяет, какие значения может принимать переменная и какие операции с ними допустимы.

Статическая типизация — типы проверяются на этапе компиляции. Пример: Java. Тип переменной фиксирован и не может изменяться.

Плюсы:

  • ошибки типов обнаруживаются до запуска программы;
  • код легче сопровождать и масштабировать;
  • повышается надежность больших систем.

Минусы:

  • требуется заранее описывать типы;
  • ниже гибкость и скорость написания прототипов.

Динамическая типизация — типы определяются во время выполнения программы.

Плюсы:

  • меньше кода, высокая гибкость;
  • удобно для прототипирования и скриптов.

Минусы:

  • ошибки типов выявляются в рантайме;
  • сложнее поддерживать крупные проекты.

Парадигмы программирования и Java

Парадигма программирования — это подход к построению программы.

  1. Объектно-ориентированная (основная): программа строится из объектов. Поддерживаются инкапсуляция, наследование и полиморфизм.
  2. Императивная (процедурная): инструкции, циклы, условия, изменение состояния.
  3. Функциональная (частично): с Java 8 — лямбда-выражения, функциональные интерфейсы и Stream API.

Итог: Java — язык высокого уровня со статической и сильной типизацией, ориентированный на ООП, при этом поддерживает императивный стиль и элементы функционального программирования.

Java: архитектура выполнения — JVM, JDK, компиляция и интерпретация

Что такое JDK (Java Development Kit)

Это набор инструментов для программиста:

  • компилятор javac
  • JVM
  • стандартные библиотеки
  • инструменты разработки
  • без JDK нельзя писать и компилировать Java

Что такое JVM (Java Virtual Machine)

Это виртуальная машина, которая выполняет Java-программу. Она:

  • запускает байт-код
  • управляет памятью
  • работает одинаково на Windows / Linux / macOS

Write once — run anywhere

Компиляция в Java

Ты пишешь код (.java). Компилятор javac компилирует его — получается байт-код (.class).

Это не машинный код, а код для JVM.

Интерпретация в Java

Что происходит при запуске:

  • JVM читает байт-код
  • выполняет его построчно (интерпретация)
  • часто используемый код дополнительно компилируется JIT

Поэтому Java одновременно:

  • и компилируемая
  • и интерпретируемая

Полный путь выполнения:

.java → javac → .class (байт-код) → JVM → выполнение
